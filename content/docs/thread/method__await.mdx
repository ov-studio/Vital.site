---
title: self:await
description: Suspends the thread until the given promise settles, then returns its resolved values
badge: Shared
---

### Syntax

```lua
local bool, ... = self:await(
    promise
)
```

<Callout type="info">
  **This function must be called from within a running thread.** It cannot be used outside a thread context, while the thread is sleeping, or while it is already awaiting another promise.

  - **Resolved** — resumes the thread with `true` followed by the resolved values
  - **Rejected** — resumes the thread with `false` followed by the rejection values
  - **Already settled** — returns immediately without yielding
</Callout>

### Parameters

| Type | Name | Description |
|------|------|-------------|
| `promise` | promise | Promise instance to await |

### Returns

| Type | Name | Description |
|------|------|-------------|
| `bool` | resolved | `true` if the promise resolved, `false` if it was rejected |
| `any` | ... | Values the promise was settled with |

---

### Examples

```lua lineNumbers title="Await a promise and print the resolved values"
local p = promise.create()

local ctx = thread.create(function(self)
    local ok, result = self:await(p)
    engine.print("info", "Resolved:", ok, result)
end)

ctx:resume()
p:resolve("done")
```

```lua lineNumbers title="Handle both resolve and reject outcomes"
local p = promise.create()

local ctx = thread.create(function(self)
    local ok, value = self:await(p)

    if ok then
        engine.print("info", "Success:", value)
    else
        engine.print("error", "Failed:", value)
    end
end)

ctx:resume()
p:reject("something went wrong")
```

---
title: exports.register
description: Registers an exported function for the calling resource
badge: Shared
---

### Syntax

```lua
local status = exports.register(
    name,
    exec
)
```

<Callout type="idea">
  **Exports allow resources to expose functions to other resources.**

  - **Scoped to the resource** — the export is automatically tied to the resource that calls `exports.register`; it cannot be registered from outside a resource environment
  - **Overwrite behaviour** — registering an export under a name that already exists will silently replace the previous function
  - **Lifecycle** — all exports registered by a resource are automatically cleaned up when that resource stops
</Callout>

### Parameters

| Type | Name | Description |
|------|------|-------------|
| `string` | name | Unique name to identify this export within the resource |
| `function` | exec | Function to expose; called when another resource invokes `exports.call` with this name |

### Returns

| Type | Name | Description |
|------|------|-------------|
| `bool` | status | `true` on successful registration, or `false` on failure |

<hr/>

### Examples

```lua lineNumbers title="Register a simple export"
exports.register("get_player_count", function()
    return 42
end)
```

```lua lineNumbers title="Register an export that accepts arguments"
exports.register("add", function(a, b)
    return a + b
end)
```

```lua lineNumbers title="Register multiple exports from the same resource"
exports.register("greet", function(name)
    return "Hello, " .. name .. "!"
end)

exports.register("farewell", function(name)
    return "Goodbye, " .. name .. "!"
end)
```

```lua lineNumbers title="Overwrite an existing export"
exports.register("get_status", function()
    return "initialising"
end)

-- Later in the same resource, replace it with a live value:
exports.register("get_status", function()
    return "running"
end)
```

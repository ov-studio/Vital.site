---
title: self:delete
description: Appends a row deletion to the query
badge: Server
---

### Syntax

```lua
local query = self:delete()
```

<Callout type="info">
  **This function only prepares the query and does not execute it.** Chain [`db_query:execute`](/docs/db_query/method/execute) at the end to commit the query to the database.

  - **Chaining** — compatible with `:where` to target specific rows for deletion
  - **Multiple deletions** — chain multiple `:delete` calls to append additional row deletions
  - **Multiple conditions** — chain multiple `:where` calls to apply additional filters to the deletion
  - **Without conditions** — omitting `:where` will delete all rows in the table
</Callout>

### Parameters

| No parameters are accepted by this function |
|---------------------------------------------|

### Returns

| Type | Name | Description |
|------|------|-------------|
| `db_query` / `bool` | query | Query builder instance on successful execution, or `false` on failure |

---

### Examples

```lua lineNumbers title="Delete a specific player record"
local db = database.create("127.0.0.1", "root", "", "vital_sandbox")

thread.create(function(self)
    local ok, err = self:await(db:table("players")
        :delete()
        :where("name", "=", "aviril")
        :execute())

    if not ok then
        engine.print("error", err)
    end
end):resume()
```

```lua lineNumbers title="Delete all rows in the 'players' table"
local db = database.create("127.0.0.1", "root", "", "vital_sandbox")

thread.create(function(self)
    local ok, err = self:await(db:table("players")
        :delete()
        :execute())

    if not ok then
        engine.print("error", err)
    end
end):resume()
```

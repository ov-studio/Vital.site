---
title: db_query:select
description: Specifies the columns to retrieve when executing a fetch query
badge: Server
---

### Syntax
```lua
local query = db_query:select(...)
```

<Callout type="info">
  **This function only prepares the query and does not execute it.** Chain [`db_query:fetch`](/docs/database/method/fetch) at the end to retrieve the results.

  - **With columns** — only the specified columns are included in the result
  - **Without columns** — all columns are returned, equivalent to `SELECT *`
  - **Multiple selections** — chain multiple `:select` calls to append additional columns to the selection
  - **Chaining** — compatible with `:where` to apply filters to the selection
</Callout>

### Parameters

| Type | Name | Description |
|------|------|-------------|
| `string` | ... | One or more column names to retrieve from the table<br/>If omitted, all columns are returned |

### Returns

| Type | Name | Description |
|------|------|-------------|
| `db_query` / `bool` | query | Query builder instance on successful execution, or `false` on failure |

<hr/>

### Examples
```lua
local db = database.create("127.0.0.1", "root", "", "vital_sandbox")

--Fetch specific columns from the players table
local rows = db:table("players")
    :select("id", "name", "score")
    :fetch()

--Fetch all columns from the players table
local rows = db:table("players")
    :fetch()
```

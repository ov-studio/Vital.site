---
title: self:reject
description: Rejects the promise with the given reason, resuming any threads awaiting it with a failure
badge: Server
---

### Syntax

```lua
local status = self:reject(
    ...
)
```

<Callout type="info">
  **A promise can only be settled once.** Calling `self:reject` or [`self:resolve`](/docs/promise/method/resolve) on an already-settled promise has no effect and returns `false`.

  - **Pending** — the promise is rejected and all awaiting threads are resumed with the provided rejection values
  - **Already settled** — the call is ignored and `false` is returned
</Callout>

### Parameters

| Type | Name | Description |
|------|------|-------------|
| `any` | ... | One or more values describing the rejection reason |

### Returns

| Type | Name | Description |
|------|------|-------------|
| `bool` | status | `true` if the promise was successfully rejected, `false` if it was already settled |

---

### Examples

```lua lineNumbers title="Reject a promise with an error message"
local p = promise.create()

p:reject("something went wrong")
```

```lua lineNumbers title="Reject a promise and handle the error from a thread"
local p = promise.create()

p:reject("connection timed out")

thread.create(function(self)
    local ok, err = self:await(p)

    if not ok then
        engine.print("error", err) --'connection timed out'
    end
end):resume()
```

```lua lineNumbers title="Attempt to reject an already-settled promise"
local p = promise.create()

p:resolve(true)
engine.print("info", p:reject("too late")) --false
```

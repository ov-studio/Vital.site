## Overview

Every **Vital.server** ships with a `config.yaml` in its root directory, pre-populated with sensible defaults. It is the single source of truth for runtime behaviour — modify the values, restart the server, and changes take effect immediately.

If the config file is absent or malformed, the server falls back to built-in defaults and emits a diagnostic to the console. A missing file is non-fatal.

<Callout type="info">
  **Where the config lives**

  - **Fixed location** — `config.yaml` must sit at the root of your `Vital.server/` directory
  - **Single file** — all sections (`server`, `network`, `http`, `discord`, `social`) live in this one file
  - **Restart to apply** — changes are not hot-reloaded; a server restart is required for any edit to take effect
</Callout>

```txt title="Example server layout"
Vital.server/
├── config.yaml
└── resources/
    └── ...
```

## Configuration

Default configuration as shipped with every Vital.server.

```yaml title="Vital.server/config.yaml"
# =============================================================================
# Vital Sandbox — Server Configuration
# =============================================================================

server:
  name: "Vital Sandbox Server"
  version: "1.0.0"
  description: "A Vital Sandbox server"

network:
  port: 7777
  max_peers: 32

http:
  port: 7778

discord:
  enabled: false
  application_id: "0"
  state: "Playing on {server_name}"
  details: "{player_count}/{max_players} players"
  large_image_key: ""
  large_image_text: ""
  small_image_key: ""
  small_image_text: ""

social:
  discord_invite: ""
  website: ""
```

<Callout type="warn">
  **All keys are optional.**

  - Every key has a built-in default and can be omitted without error
  - An absent or malformed `config.yaml` is non-fatal — the server starts with defaults and logs a warning
</Callout>

#### Server

Controls the display name, version string, and description exposed to clients and integrations.

```yaml
server:
  name: "Vital Sandbox Server"
  version: "1.0.0"
  description: "A Vital Sandbox server"
```

| Key | Type | Default | Description |
|---|---|---|---|
| `name` | `string` | `"Vital Sandbox Server"` | Display name shown in the master-list and Discord Rich Presence |
| `version` | `string` | `"1.0.0"` | Free-text version identifier — not validated or compared |
| `description` | `string` | `""` | Optional description shown in the master-list |

---

#### Network

Controls the ENet multiplayer transport layer. Clients connect to the port defined here.

```yaml
network:
  port: 7777
  max_peers: 32
```

<Callout type="warn">
  **Both values directly affect client connectivity — change them with care.**

  - **`port`** — clients and any firewall or port-forwarding rules must be updated to match
  - **`max_peers`** — once the limit is reached, incoming connections are refused until a slot becomes available
</Callout>

| Key | Type | Default | Description |
|---|---|---|---|
| `port` | `integer` | `7777` | UDP/TCP port the ENet multiplayer server listens on |
| `max_peers` | `integer` | `32` | Maximum number of concurrent client connections |

---

#### HTTP

Controls the asset delivery server. Clients fetch assets such as models and textures from this port at runtime.

```yaml
http:
  port: 7778
```

<Callout type="info">
  **The HTTP server operates independently of the ENet transport layer.**

  - **Both ports must be reachable** — blocking either one will result in a failed or degraded session
  - **Separate firewall rules** — ensure both ports are open and forwarded correctly in your network configuration
</Callout>

| Key | Type | Default | Description |
|---|---|---|---|
| `port` | `integer` | `7778` | Port the HTTP asset delivery server listens on |

#### Discord

Configures Discord Rich Presence. This section **only applies to client builds** — it has no effect on server-only deployments.

```yaml
discord:
  enabled: false
  application_id: "0"
  state: "Playing on {server_name}"
  details: "{player_count}/{max_players} players"
  large_image_key: ""
  large_image_text: ""
  small_image_key: ""
  small_image_text: ""
```

<Callout type="info">
  **Image asset keys must be registered before use.**

  - **Register assets** — upload and name your images in the [Discord Developer Portal](https://discord.com/developers/applications) under **Rich Presence → Art Assets**
  - **Application ID** — non-numeric or empty values are treated as `0`, and Rich Presence will not initialize
  - **Placeholders** — `state` and `details` support runtime tokens resolved at startup (see table below)
</Callout>

| Key | Type | Default | Description |
|---|---|---|---|
| `enabled` | `boolean` | `false` | Enables or disables Discord Rich Presence entirely |
| `application_id` | `string` | `"0"` | Application ID from the Discord Developer Portal - must be a numeric string |
| `state` | `string` | `"Playing on {server_name}"` | State line displayed below the game name in Discord |
| `details` | `string` | `"{player_count}/{max_players} players"` | Details line displayed above the state in Discord |
| `large_image_key` | `string` | `""` | Asset key for the large Rich Presence image |
| `large_image_text` | `string` | `""` | Tooltip text shown on hover over the large image |
| `small_image_key` | `string` | `""` | Asset key for the small Rich Presence image |
| `small_image_text` | `string` | `""` | Tooltip text shown on hover over the small image |

The `state` and `details` fields support the following runtime placeholders, resolved each time presence is updated:

| Placeholder | Resolves To |
|---|---|
| `{server_name}` | Value of `server.name` |
| `{player_count}` | Current number of connected players |
| `{max_players}` | Value of `network.max_peers` |

---

#### Social

Defines community links surfaced in the master-list.

```yaml
social:
  discord_invite: ""
  website: ""
```

| Key | Type | Default | Description |
|---|---|---|---|
| `discord_invite` | `string` | `""` | Discord server invite URL shown in the master-list |
| `website` | `string` | `""` | Community or project website URL shown in the master-list |
